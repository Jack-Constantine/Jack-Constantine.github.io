📄 AutoFigure-Edit: Generating Editable Scientific Illustrations via Reference-Guided Styling

一、问题定义

  • 真实要解决的问题：把长篇方法文本直接变成可编辑、可局部修改的科研插图，而不是只能得到一次性 raster 图。
  • 为什么这个问题重要：科研插图通常要花几天，并且后续修改频繁；如果输出不可编辑，研究者会被重新作图成本拖死。
  • 现有方法为什么失败：T2I 模型视觉上可观但结构不稳；纯 SVG/代码法可编辑但布局和美观性差；PNG 转 SVG 会继承上游结构错误。

二、方法机制

X → M → Y

  • 输入是什么：长上下文科研文本 + 参考风格图。
  • 核心机制是什么：四阶段流水线
    • Stage I：风格条件图像生成，先得到 raster 草图
    • Stage II：分割与结构索引，把图拆成可定位部件
    • Stage III：资产提取，保留可复用视觉元素
    • Stage IV：SVG 模板生成与二次 refinement，把结构映射成可编辑 SVG
  • 输出是什么：带嵌入式可视化编辑器的可编辑 SVG 插图。

  • 是否可公开下载：

    • 这一批整体上 retrieval 不是最强主线
  • benchmark类
    • FigureBench
    • CRAFTBENCH
    • VisBench
    • SridBench

2. 共性模块库

  • harness / orchestration layer
  • 中间规范层（TikZ / figure specification / SVG skeleton / MCP message）
  • critic / refinement loop
  • 可编辑输出层（TikZ / SVG / Visio objects）
  • benchmark with multi-axis scoring
  • 结构先验或版面约束

3. 矛盾点

  • 更强 backbone vs 更强 harness
    • CRAFTER、chart-plot 明确主张“不是更强模型，而是更强外层 harness”
    • AutomaTikZ 更像“让模型适配更合适的图形语言”
  • 可编辑性优先 vs 视觉丰富度优先
    • TikZ / VisPainter / CRAFTEDITOR 偏可编辑
    • AUTOFIGURE 偏 publication-ready 视觉表现
  • benchmark 目标不同
    • SridBench 偏图像质量与科研绘图能力
    • VisBench 偏高密度可编辑图
    • CRAFTBENCH 偏跨图类与跨条件泛化

4. 研究空白（Research Gaps）

  • 缺少统一 benchmark 同时覆盖：科学正确性、视觉审美、可编辑性、交互成本。
  • 多数方法仍依赖闭源 API，复现性和成本不稳定。
  • 真正跨学科泛化仍不足，尤其是 biology / chemistry / medicine 的复杂规范图。
  • 中间表示尚未统一，不同系统之间难以模块级互操作。
  • 评测仍缺“长期修改可维护性”指标，例如改一处会不会牵一片。

5. 高潜力研究方向（Top 10）

    1. Harness-first scientific figure stack
    • 把 backbone 视为可替换执行器，重点优化 spec memory、critic、verification。
    1. Unified figure DSL
    • 统一 TikZ / SVG / DrawIO / Visio 的中间结构表示。
    1. Editable benchmark unification
    • 合并 VisBench、CRAFTBENCH、SridBench、FigureBench 的优点做总 benchmark。
    1. Scientific chart + scientific figure fusion
    • 把 chart-plot 的投稿级图表能力并入更大的科研图系统。
    1. MCP-native figure agents
    • 像 VisPainter 一样，把专业图形软件直接纳入 agent 执行层。
    1. Structure-aware raster-to-vector editing
    • 在 CRAFTEDITOR 基础上进一步提高图元级可修改性。
    1. Domain-specialized scientific figure generation
    • 针对 biology、chemistry、materials 建专门结构库和校验器。
    1. Compile-and-verify graphics generation
    • 借鉴 AutomaTikZ 的 compile feedback 与 GeoSVG-RL 的几何奖励，做统一 verifier。
    1. Cost-aware agent design
    • 在保持质量的前提下减少多轮 API 迭代成本。
    1. Human-in-the-loop controllable editing
    • 把自动生成与人工微调做成连续工作流，而不是两张皮。
